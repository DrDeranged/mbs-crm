## 1B. HTML-escape template variable values

In artifacts/api-server/src/lib/renderPdf.ts, renderTemplate inserts user-supplied
fieldValues into HTML with NO escaping. Replace the function so VALUES are escaped
while template HTML itself stays raw:

function escapeHtml(value: string): string {
  return value
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&#39;");
}

export function renderTemplate(htmlTemplate: string, fieldValues: Record<string, string>): string {
  return htmlTemplate.replace(/\{\{(\w+)\}\}/g, (_, key) => {
    const raw = fieldValues[key];
    return raw == null ? "" : escapeHtml(String(raw));
  });
}

renderTemplate is also imported by routes/email.ts and lib/dripJob.ts — this shared
fix is correct for both (lead names/companies in email variables must be escaped too).

export async function generateLeadBriefing(leadId: number): Promise<string>
- Builds the context bundle
- Calls Claude with system prompt:
  "You are a sales intelligence assistant for an MCA/equipment-financing brokerage.
   Given the lead data below, write a briefing for the sales rep with exactly these
   sections: SNAPSHOT (2-3 sentences on who this is and deal viability),
   FINANCIAL PICTURE (2-3 sentences), ENGAGEMENT (what's happened so far, 1-2 sentences),
   RISKS (bullet-style, max 3), NEXT BEST ACTIONS (numbered, max 3, specific and
   immediately actionable). Be concrete, use the numbers provided, no filler,
   under 250 words total."
- max_tokens: 600
- Saves the result to leads.aiSummary + aiSummaryAt, returns it

export async function generateDraft(leadId: number, channel: "email" | "sms", instruction?: string): Promise<{ subject?: string; body: string }>
- Builds the same context bundle
- System prompt: "You draft outbound {channel} messages for a business-financing sales
  rep. Write in a professional, warm, concise tone. Never invent facts, numbers, rates,
  or approval promises not present in the context. Never mention AI. For email return
  JSON {\"subject\": string, \"body\": string}; for SMS return JSON {\"body\": string}
  with body under 300 characters. Return ONLY valid JSON."
- Include the rep's optional instruction (e.g. "nudge about missing bank statements")
  in the user message
- max_tokens: 500; parse JSON defensively (strip ```json fences if present)
- Returns the draft. NEVER sends anything — drafts only.

export async function runRenewalJob(): Promise<void>
- Query leads where: status = "funded", fundedAt IS NOT NULL, renewalFlaggedAt IS NULL
- For each: termDays = estimatedTermDays ?? 180 (default 6 months when unknown);
  elapsed = days since fundedAt; if elapsed >= 0.6 * termDays:
  - Set renewalFlaggedAt = now
  - createNotification to the assigned rep: type "renewal_due", title "Renewal opportunity",
    body "{Lead/Company name} is ~{pct}% through their estimated term — time to discuss re-funding"
  - notifyAllManagers with the same
  - logActivity action "renewal_flagged" with details { elapsedDays, termDays }
- Wrap the whole run in try/catch with logger.error — the job must never crash the server

Schedule it in artifacts/api-server/src/index.ts next to the existing drip/task-reminder
